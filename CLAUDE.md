# Tử Vi Việt Nam — AI Astrology App

## Project Overview
Vietnamese astrology app combining traditional Tử Vi (紫微斗數) fortune-telling with AI. Built with Lovable.dev, deployed to web. Now migrating to mobile via Capacitor.

## Tech Stack
- **Frontend**: React 18 + TypeScript + Vite + Tailwind CSS + shadcn/ui
- **Backend**: Supabase (Auth, PostgreSQL, Edge Functions, Realtime)
- **Astrology Engine**: iztro v2.5.7 (紫微斗數 calculations)
- **AI**: Claude API via Supabase Edge Function (`analyze-chart`)
- **Payments**: VietQR bank transfer + SePay webhook auto-verification
- **State**: React Query + React Context (AuthContext)
- **Routing**: react-router-dom v6 (BrowserRouter)

## Commands
- `npm run dev` — Start Vite dev server on port 8080
- `npm run build` — Production build to `dist/`
- `npm run test` — Run Vitest tests
- `npm run lint` — ESLint check

## Architecture
```
src/
├── pages/           # Route pages (Index, Auth, BoiKieu, BoiQue, VanHan, etc.)
├── components/      # UI components + shadcn/ui primitives
├── contexts/        # AuthContext (Supabase auth wrapper)
├── hooks/           # useStreamingAnalysis, useFeatureAccess, useChartAccess, etc.
├── lib/tuvi/        # Core Tử Vi logic (tuviChart, lunarCalendar, starMeanings, etc.)
├── integrations/    # Supabase client + generated types
├── services/        # TuViService
├── utils/           # vietqr.ts (QR generation, pricing, transfer codes)
supabase/
├── functions/       # Edge Functions (analyze-chart, admin-actions)
├── migrations/      # SQL migrations
```

## Key Files
- `src/App.tsx` — Router with 13 routes, uses BrowserRouter
- `src/components/VietQRPaymentModal.tsx` — Payment flow (QR → pending → verify → success)
- `src/contexts/AuthContext.tsx` — Supabase auth with profile fetching
- `src/integrations/supabase/client.ts` — Supabase client (auto-generated by Lovable)
- `src/utils/vietqr.ts` — VietQR URL generation, pricing config, transfer code generation
- `supabase/functions/analyze-chart/index.ts` — Claude API integration with CORS whitelist

## Database (20+ tables)
Key tables: `profiles`, `payments`, `chart_analyses`, `kieu_packages`, `boi_que_packages`, `van_han_packages`, `luan_giai_packages`, `user_features`, `minted_nfts`, `api_rate_limits`

## Important Patterns
- **Payment flow**: VietQRPaymentModal creates pending record on QR display → SePay webhook auto-verifies → Supabase Realtime subscription updates UI
- **Feature access**: Package-based (3 uses per 39,000đ purchase), NOT subscription
- **CORS**: Edge Functions have strict origin whitelist in `ALLOWED_ORIGINS` array
- **Auth**: Supabase email/password, redirect URL uses `window.location.origin`
- **localStorage**: Used for payment pending state persistence (storageKey pattern)

## Critical Rules
- **NEVER modify `src/integrations/supabase/client.ts`** — auto-generated by Lovable, will be overwritten
- **NEVER modify `src/integrations/supabase/types.ts`** — auto-generated by Lovable
- Edge Function CORS changes MUST also be applied in Lovable.dev to survive redeploy
- Vietnamese language UI — all user-facing text must be in Vietnamese
- Bank account info in `VietQRPaymentModal.tsx` and `vietqr.ts` must stay in sync
- Transfer codes must include random characters to prevent false positive payment matching

## Current Task: Capacitor Mobile Migration
Converting this Lovable.dev web app to native Android (Google Play) and iOS (App Store) using Capacitor.

### What Needs to Change
1. **Router**: BrowserRouter → conditional MemoryRouter for native, keep BrowserRouter for web
2. **External links**: All `window.open()` calls → Capacitor Browser plugin
3. **CORS**: Add `capacitor://localhost` (iOS) and `http://localhost` (Android) to Edge Function ALLOWED_ORIGINS
4. **Auth redirects**: `window.location.origin` → native URL scheme for Capacitor
5. **index.html**: Add viewport-fit=cover, theme-color, apple-mobile-web-app meta tags
6. **CSS**: Add safe-area-inset CSS variables for notch/Dynamic Island
7. **Capacitor config**: appId=vn.tuviapp.app, dark theme, splash screen matching brand

### What Should NOT Change
- Supabase client configuration
- Payment logic (VietQR + SePay webhook)
- iztro calculations
- AI analysis edge function logic
- Database schema
- Component business logic
